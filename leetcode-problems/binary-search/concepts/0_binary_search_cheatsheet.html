Fundamentals & Mental Model

What Is Binary Search?

A divide-and-conquer strategy that repeatedly halves a search space. Works on any monotone predicate — not just sorted arrays. The core idea:

💡
At every step, you eliminate half of the remaining candidates with certainty. You never revisit eliminated space. This gives O(log n) time.
TimeO(log n)
SpaceO(1) iterative · O(log n) recursive
Iterations max⌈log₂(n)⌉ ≈ 30 for n=10⁹

The Two Core Questions

Before writing code, answer:

  1. Q1
    What is my search space?
    Indices [lo..hi] into array? Or a value range [min_val..max_val]? Know this before lo/hi setup.
  2. Q2
    What is my predicate?
    A boolean function f(x) such that the space looks like [F,F,F,...,T,T,T]. You want the boundary.
🎯
Every binary search is finding where a predicate flips from False → True (or True → False).

Prerequisites / Applicability Checklist

✅ You CAN use BS when:

    • Array is sorted (ascending or descending)
    • Search space has a monotone property
    • Function feasible(x) is monotone
    • You can define lo/hi bounds clearly
    • "Find min X such that condition holds"
    • Rotated sorted array (with adaptation)

❌ BS does NOT work when:

    • Array is unsorted (no monotonicity)
    • Multiple separate valid ranges
    • Predicate is not monotone (oscillates)
    • You need all occurrences (use BS + expand)

⚠️ Handle carefully:

    • Integer overflow in mid computation
    • lo/hi initialization (in-bounds vs sentinel)
    • Inclusive vs exclusive boundaries
    • Off-by-one in the return value
    • Empty arrays → lo > hi on entry

The 3 Templates (Memorise These)

⚡
Pick ONE template and master it. Template 1 (closed interval) is the most versatile and recommended for interviews. Templates 2 & 3 are for specific boundary-finding patterns.

Template 1 — Exact Match / Closed Interval [lo, hi]

RECOMMENDED
Python
def binary_search(nums, target):
    lo, hi = 0, len(nums) - 1   # both INCLUSIVE
    while lo <= hi:              # ≤ (processes single element)
        mid = lo + (hi - lo) // 2  # no overflow
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            lo = mid + 1             # mid ruled out
        else:
            hi = mid - 1             # mid ruled out
    return -1                       # not found
Java / C++
int binarySearch(int[] nums, int target) {
    int lo = 0, hi = nums.length - 1;
    while (lo <= hi) {
        // Use unsigned right shift to avoid overflow
        int mid = lo + (hi - lo) / 2;
        // OR: int mid = (lo + hi) >>> 1; (Java)
        if (nums[mid] == target) return mid;
        if (nums[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}
✅
Key invariant: At exit (lo > hi), every element in [0..lo-1] is < target, every element in [hi+1..n-1] is > target. Loop terminates because search space shrinks every iteration (mid ± 1 always).

Template 2 — Find First True / Left Boundary [lo, hi) — half-open

hi starts at n (one past end). Loop condition is lo < hi. At exit, lo == hi == first index where predicate is True. If no True, lo == n.

Python — bisect_left equivalent
def lower_bound(nums, target):
    lo, hi = 0, len(nums)  # hi = n (exclusive)
    while lo < hi:           # strict <
        mid = lo + (hi - lo) // 2
        if nums[mid] < target:
            lo = mid + 1
        else:               # nums[mid] >= target
            hi = mid        # keep mid as candidate
    return lo              # first index where nums[i] >= target
General Predicate Form
def first_true(lo, hi, predicate):
    # finds first x in [lo,hi] where pred(x) is True
    # pred must be: F,F,...,T,T,T
    while lo < hi:
        mid = lo + (hi - lo) // 2
        if predicate(mid):
            hi = mid    # could be answer, look left
        else:
            lo = mid + 1  # not answer, discard
    return lo

Template 3 — Find Last True / Right Boundary

hi starts at n-1 (inclusive). At exit, lo == hi == last index where predicate is True. Tricky: mid must bias toward hi to avoid infinite loop. Use mid = lo + (hi - lo + 1) // 2 (ceiling division).

Python — bisect_right equivalent
def upper_bound(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo < hi:
        # ceiling mid — prevents infinite loop!
        mid = lo + (hi - lo + 1) // 2
        if nums[mid] <= target:
            lo = mid      # keep mid as candidate
        else:
            hi = mid - 1
    return lo              # last index where nums[i] <= target
General Last-True Form
def last_true(lo, hi, predicate):
    # pred must be: T,T,...,F,F
    while lo < hi:
        # ⚠️ ceiling mid is CRITICAL here
        mid = lo + (hi - lo + 1) // 2
        if predicate(mid):
            lo = mid    # keep mid as candidate
        else:
            hi = mid - 1
    return lo
🔥
Why ceiling mid for right boundary? When lo=4, hi=5: floor mid=4. If predicate(4)=True → lo=4, hi=5 unchanged → INFINITE LOOP. Ceiling mid=5 always advances lo or shrinks hi.

Loop Invariants & Termination Proofs

Invariant for Template 1 (lo ≤ hi)

  • Before loop: target ∈ nums[lo..hi] if it exists
  • Each iteration: mid divides [lo,hi] into [lo,mid-1], mid, [mid+1,hi]
  • Update lo=mid+1: target ∉ [lo..mid] → narrows down
  • Update hi=mid-1: target ∉ [mid..hi] → narrows down
  • Termination: lo > hi means empty interval — target absent
📐
Search space shrinks by at least 1 every iteration (mid ± 1). So terminates in at most log₂(n+1) steps.

Invariant for Templates 2 & 3 (lo < hi)

  • Invariant: answer ∈ [lo, hi] always
  • Template 2: first True ∈ [lo, hi]. When pred(mid)=True: hi=mid (answer could be mid). When False: lo=mid+1 (mid is not answer)
  • Template 3: last True ∈ [lo, hi]. When pred(mid)=True: lo=mid (mid could be answer). When False: hi=mid-1
  • Termination: lo==hi → exactly one candidate left
  • Why ceil for T3: lo=mid with floor would stall when hi=lo+1

Mid Calculation Deep Dive

# ❌ WRONG — integer overflow in Java/C++
mid = (lo + hi) // 2
# If lo=hi=2^31-1, lo+hi overflows 32-bit int!

# ✅ CORRECT — no overflow
mid = lo + (hi - lo) // 2      # floor (default)
mid = lo + (hi - lo + 1) // 2  # ceiling

# Java only — unsigned right shift:
// int mid = (lo + hi) >>> 1;  // safe even with overflow

# Python — no overflow (arbitrary precision), but
# still best practice to use lo + (hi-lo)//2
# Summary of floor vs ceiling bias:
#
# floor(lo=0, hi=1) → mid=0  (biased left)
# ceil (lo=0, hi=1) → mid=1  (biased right)
#
# Template 2 (lo=mid+1 or hi=mid):
#   floor is SAFE because lo=mid+1 always advances
#
# Template 3 (lo=mid or hi=mid-1):
#   floor causes INFINITE LOOP when lo+1==hi
#   because lo=mid=lo, nothing changes!
#   ceiling REQUIRED → mid=hi, lo advances to hi

All Variants with Code

4.1 First & Last Occurrence (Search Range)

LC 34 
def search_range(nums, target):
    def first():
        lo, hi = 0, len(nums)
        while lo < hi:
            mid = lo + (hi - lo) // 2
            if nums[mid] < target: lo = mid + 1
            else: hi = mid
        return lo if lo < len(nums) and nums[lo] == target else -1

    def last():
        lo, hi = 0, len(nums)
        while lo < hi:
            mid = lo + (hi - lo) // 2
            if nums[mid] <= target: lo = mid + 1  # ≤ (not just <)
            else: hi = mid
        return lo - 1 if lo > 0 and nums[lo-1] == target else -1

    return [first(), last()]

# Python shortcut using bisect:
import bisect
left = bisect.bisect_left(nums, target)
right = bisect.bisect_right(nums, target) - 1
# Validate: nums[left]==target (else target absent)

4.2 Search in Rotated Sorted Array

LC 33LC 81 (dups)

Key insight: one half is always sorted after any rotation. Check which half is sorted, then determine if target lies in that sorted half.

def search(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = lo + (hi - lo) // 2
        if nums[mid] == target: return mid

        # Is LEFT half sorted?
        if nums[lo] <= nums[mid]:  # ≤ handles duplicates
            # Target in left sorted range?
            if nums[lo] <= target < nums[mid]:
                hi = mid - 1
            else:
                lo = mid + 1
        else:  # RIGHT half is sorted
            if nums[mid] < target <= nums[hi]:
                lo = mid + 1
            else:
                hi = mid - 1
    return -1

# LC 81 (with duplicates): add edge case at start of loop:
# if nums[lo] == nums[mid] == nums[hi]: lo++; hi-- → O(n) worst case
⚠️
Find Rotation Index: Find the index of the minimum element using BS on answer — predicate: nums[mid] < nums[0].

4.3 Find Peak Element

LC 162

A peak is where nums[i] > nums[i-1] and nums[i] > nums[i+1]. Treat nums[-1]=nums[n]=-∞.

def findPeakElement(nums):
    lo, hi = 0, len(nums) - 1
    while lo < hi:
        mid = lo + (hi - lo) // 2
        if nums[mid] < nums[mid + 1]:
            lo = mid + 1  # ascending → peak is to the right
        else:
            hi = mid      # descending/equal → peak is here or left
    return lo

# Why this works: if nums[mid] < nums[mid+1], the right slope guarantees
# at least one peak exists to the right (boundary is -∞). Symmetric logic for left.

4.4 Search a 2D Matrix

LC 74LC 240

LC 74 — Row-major sorted (last of row < first of next). Treat as 1D array:

def searchMatrix(matrix, target):
    m, n = len(matrix), len(matrix[0])
    lo, hi = 0, m * n - 1
    while lo <= hi:
        mid = lo + (hi - lo) // 2
        # Convert 1D index to 2D
        val = matrix[mid // n][mid % n]
        if val == target: return True
        if val < target: lo = mid + 1
        else: hi = mid - 1
    return False

LC 240 — Each row & col sorted. Start top-right:

def searchMatrix240(matrix, target):
    if not matrix: return False
    r, c = 0, len(matrix[0]) - 1
    while r < len(matrix) and c >= 0:
        if matrix[r][c] == target:
            return True
        elif matrix[r][c] > target:
            c -= 1   # eliminate column
        else:
            r += 1   # eliminate row
    return False
# O(m+n) — not BS but same elimination logic

4.5 Integer Square Root / nth Root

LC 69 
def mySqrt(x):
    if x < 2: return x
    lo, hi = 1, x // 2          # sqrt(x) ≤ x//2 for x≥4
    while lo < hi:
        mid = lo + (hi - lo + 1) // 2  # ceiling — finding LAST True
        if mid * mid <= x:
            lo = mid
        else:
            hi = mid - 1
    return lo
# Predicate: mid² ≤ x → find LAST True → ceiling mid template

# Floating point sqrt (more precision):
def mySqrtF(x, eps=1e-9):
    lo, hi = 0.0, max(1.0, float(x))
    while hi - lo > eps:       # terminate on precision
        mid = (lo + hi) / 2
        if mid * mid <= x: lo = mid
        else: hi = mid
    return lo

Binary Search on the Answer (Most Powerful Pattern)

🚀
The big idea: Instead of searching an array, search the answer space directly. Ask: "Is it possible to achieve answer ≤ X?" — if monotone (possible for X → possible for X+1), use BS to find the minimum feasible X.

Universal BS-on-Answer Template

def solve(...):
    def feasible(x) -> bool:
        # Return True if answer x is achievable / condition holds
        # This must be MONOTONE: if feasible(x) then feasible(x+1)
        ...

    lo, hi = min_possible_answer, max_possible_answer
    while lo < hi:
        mid = lo + (hi - lo) // 2
        if feasible(mid):
            hi = mid        # mid works, try smaller (find MINIMUM feasible)
        else:
            lo = mid + 1    # mid too small, need larger
    return lo  # smallest x where feasible(x) is True

# For MAXIMUM feasible x (find last True):
# if feasible(mid): lo = mid (ceiling!) else: hi = mid - 1

Example: Koko Eating Bananas (LC 875)

def minEatingSpeed(piles, h):
    def feasible(speed):
        return sum((p - 1) // speed + 1 for p in piles) <= h
        # ceiling division: math.ceil(p/speed)

    lo, hi = 1, max(piles)
    while lo < hi:
        mid = lo + (hi - lo) // 2
        if feasible(mid): hi = mid
        else: lo = mid + 1
    return lo

Example: Split Array Largest Sum (LC 410)

def splitArray(nums, k):
    def feasible(limit):
        parts, curr = 1, 0
        for n in nums:
            if curr + n > limit:
                parts += 1
                curr = 0
            curr += n
        return parts <= k

    lo, hi = max(nums), sum(nums)
    while lo < hi:
        mid = lo + (hi - lo) // 2
        if feasible(mid): hi = mid
        else: lo = mid + 1
    return lo

All Patterns Catalogue

🎯 Pattern 1: Classic Exact Search

Sorted array, find value. Return index or -1.

  • Template 1 (lo ≤ hi)
  • Standard 3-way branch
  • Used as sub-routine in harder problems

🔎 Pattern 2: Boundary Finding

First/last occurrence, count of target, insertion position.

  • bisect_left → first occurrence
  • bisect_right - 1 → last occurrence
  • count = bisect_right - bisect_left

🌀 Pattern 3: Rotated Array

One half always sorted. Check which half, then check if target in that half.

  • Find minimum in rotated
  • Search target in rotated
  • Rotation count = index of minimum

⛰️ Pattern 4: Peak Finding

Unimodal array or bitonic. Go toward the rising slope.

  • nums[mid] < nums[mid+1] → go right
  • Mountain array peak (LC 852)
  • Find in mountain array (LC 1095)

📊 Pattern 5: Binary Search on Answer

Abstract search space. feasible(x) is monotone over answer values.

  • Min speed / capacity / time
  • Max tasks with constraints
  • Kth smallest/largest problems

🗺️ Pattern 6: 2D Binary Search

Matrix with sorted rows or row-major order. Map to 1D or staircase elimination.

  • LC 74: row-major → 1D index
  • LC 240: start top-right corner
  • Kth smallest in sorted matrix

🔢 Pattern 7: Numerical / Math BS

Integer sqrt, nth root, power functions. Search over value range.

  • Integer result → Last True template
  • Float result → epsilon termination
  • Overflow guard: use float or long

🎲 Pattern 8: Kth Element (BS + Count)

Kth smallest in sorted matrix / merged lists. Count elements ≤ mid.

  • predicate: count(≤ x) ≥ k
  • lo=min, hi=max of value range
  • Final answer is always in the array

📈 Pattern 9: Median of Two Sorted Arrays

Partition both arrays. Binary search on partition index of smaller array.

  • O(log(min(m,n))) time
  • Edge: partition at 0 or n → ±∞
  • LC 4 — hardest BS problem

🔗 Pattern 10: BS in Implicit Search Space

No array given. Search space is conceptual (e.g., number of days, workers).

  • Capacity to ship packages
  • Minimum days to bloom
  • Construct the feasible() function

🔀 Pattern 11: Parallel BS on Multiple Arrays

Count elements across multiple sorted arrays satisfying a condition.

  • Kth smallest in M sorted arrays
  • Sum of squares ≤ k
  • Use inner BS per outer iteration

🌊 Pattern 12: Floating Point BS

Answer is real-valued. Terminate when precision is sufficient.

  • Loop 50–100 times OR hi-lo < ε
  • No mid ± 1 (continuous space)
  • Bisect continuously until convergent

Classic Bugs, Gotchas & Edge Cases

🔥
Bug 1: Infinite Loop (most common)
Using hi = mid in Template 3 WITHOUT ceiling mid. When lo+1==hi, mid=lo, lo=mid=lo → nothing changes.
Fix: mid = lo + (hi - lo + 1) // 2 when doing lo = mid
💥
Bug 2: Integer Overflow
In Java/C++, (lo + hi) / 2 overflows if lo+hi > INT_MAX.
Fix: lo + (hi - lo) / 2 always.
⚠️
Bug 3: Off-by-one in hi init
Template 2 uses hi=n (exclusive). Template 1 uses hi=n-1 (inclusive). Mixing them causes out-of-bounds.
Rule: lo<hi → hi=n; lo≤hi → hi=n-1
⚠️
Bug 4: Wrong comparison for duplicates
Using > instead of ≥ (or vice versa) causes boundary to skip the correct element.
Rule: lower_bound uses strict <; upper_bound uses ≤
⚠️
Bug 5: Not validating the returned index
lower_bound returns the insertion point even if target doesn't exist. Always check: if lo < n and nums[lo] == target.
⚠️
Bug 6: BS on Answer — wrong lo/hi range
lo must include the minimum possible answer, hi the maximum. If lo is too high or hi too low, correct answer is excluded from search space.
💡
Edge: Empty array
If nums is empty: Template 1 → lo=0 > hi=-1, loop never runs, return -1. Template 2 → lo=hi=0, loop never runs, return 0. Always verify caller handles n=0.
💡
Edge: Single element array
lo=hi=0. Template 1: mid=0, processes and returns. Template 2: lo==hi, loop skips, returns 0 immediately. Handle this in test cases.
💡
Edge: All elements same
first(target) → 0, last(target) → n-1 if target matches. Rotated array with all same → LC 81 degrades to O(n); handle with lo++, hi-- when nums[lo]==nums[mid]==nums[hi].
💡
Edge: Floating point BS precision
Don't use hi - lo > 1e-9 without thinking about the range. For values near 0 use relative epsilon. Fixed 50-100 iterations is safer.

Advanced Techniques

Median of Two Sorted Arrays (LC 4) — O(log min(m,n))

def findMedianSortedArrays(A, B):
    # Always binary search on the shorter array
    if len(A) > len(B): A, B = B, A
    m, n = len(A), len(B)
    lo, hi = 0, m
    while lo <= hi:
        px = (lo + hi) // 2        # partition A at px
        py = (m + n + 1) // 2 - px  # partition B at py

        Aleft  = A[px-1] if px > 0 else float('-inf')
        Aright = A[px]   if px < m else float('inf')
        Bleft  = B[py-1] if py > 0 else float('-inf')
        Bright = B[py]   if py < n else float('inf')

        if Aleft <= Bright and Bleft <= Aright:
            # Perfect partition
            if (m + n) % 2:
                return max(Aleft, Bleft)
            return (max(Aleft,Bleft) + min(Aright,Bright)) / 2
        elif Aleft > Bright: hi = px - 1
        else: lo = px + 1

Kth Smallest in Sorted Matrix (LC 378)

def kthSmallest(matrix, k):
    n = len(matrix)
    def count_le(mid):
        # Count elements ≤ mid using staircase from bottom-left
        count, r, c = 0, n-1, 0
        while r >= 0 and c < n:
            if matrix[r][c] <= mid:
                count += r + 1  # all in this col up to r
                c += 1
            else:
                r -= 1
        return count

    lo, hi = matrix[0][0], matrix[-1][-1]
    while lo < hi:
        mid = lo + (hi - lo) // 2
        if count_le(mid) >= k: hi = mid
        else: lo = mid + 1
    return lo
# Note: lo always converges to an actual matrix value

Python bisect Module — Complete Reference

import bisect

# bisect_left(a, x, lo=0, hi=len(a))
# → leftmost i where a[i] >= x (insertion point to keep sorted)
# → use for: first occurrence, lower_bound, count of elements < x
i = bisect.bisect_left(a, x)
# a[i] == x? → x found at index i (first occurrence)
# a[i] != x? → x not in array

# bisect_right(a, x)  [same as bisect.bisect(a, x)]
# → leftmost i where a[i] > x
# → use for: last occurrence+1, upper_bound, insert after existing x
j = bisect.bisect_right(a, x)
# last occurrence of x = j - 1 (if j > 0 and a[j-1] == x)
# count of x in array = j - i

# insort_left / insort_right: insert x maintaining sorted order (O(n) due to shift)
bisect.insort_left(a, x)   # insert before existing x's
bisect.insort_right(a, x)  # insert after existing x's

# Common patterns:
count = bisect.bisect_right(a,x) - bisect.bisect_left(a,x)  # count x in a
rank  = bisect.bisect_left(a, x)                            # rank of x (0-indexed)
floor_val = a[bisect.bisect_right(a,x)-1] if a else None       # largest ≤ x
ceil_val  = a[bisect.bisect_left(a,x)]   if ... else None   # smallest ≥ x

Bonus: LIS in O(n log n) using Binary Search

def lengthOfLIS(nums):
    tails = []  # tails[i] = smallest tail element of all IS of length i+1
    for n in nums:
        # Find first tail >= n (lower_bound)
        pos = bisect.bisect_left(tails, n)
        if pos == len(tails): tails.append(n)  # extends LIS
        else: tails[pos] = n                # replaces to minimize tail
    return len(tails)
# tails is always sorted but NOT the actual LIS. Only length is correct.
# Use bisect_right for strictly decreasing IS, or non-strictly increasing variations.

Interview Checklist — Before You Code

  • Identify the search space
    Is it array indices [0..n-1] or value range [min..max]? Say it out loud to the interviewer.
  • Define the predicate
    Write out f(x) = True/False explicitly. Verify monotonicity (F...F,T...T or T...T,F...F).
  • Choose the template
    Exact match → T1. First True → T2. Last True → T3. State which you're using.
  • Set lo and hi correctly
    For T1: hi=n-1. For T2: hi=n. For T3: hi=n-1. For BS on answer: hi=max possible answer.
  • Choose mid bias
    If you write lo=mid anywhere → MUST use ceiling mid. Draw lo=4, hi=5 to verify no infinite loop.
  • Handle exit condition
    What does lo mean at exit? Validate: check if nums[lo] == target before returning lo as "found".
  • Edge cases
    Empty array → n=0. Single element. All same values. Target smaller than all / larger than all.
  • Verify no overflow
    Use lo + (hi-lo)//2, not (lo+hi)//2. Especially in Java/C++.

🎯 30-Second BS Problem Solver Framework

Step 1: Is there a monotone property? If no → sort or use different algo.
Step 2: What am I searching? Indices or values?
Step 3: What is feasible(x)? Write it as a boolean check.
Step 4: Am I finding first True or last True?
Step 5: Set lo, hi. Verify they bracket the answer.
Step 6: Write loop, mid, two branches, return lo.
Step 7: Dry-run on 2-element array to check termination.
