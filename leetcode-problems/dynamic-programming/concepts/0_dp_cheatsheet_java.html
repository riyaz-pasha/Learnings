Foundations

Core Concepts
Two Prerequisites
When Does DP Apply?
  • Optimal SubstructureOptimal solution contains optimal solutions to subproblems. Greedy also has this — the difference is DP branches over all choices.
  • Overlapping SubproblemsSame sub-computation recurs. Divide & Conquer (Merge Sort) splits into non-overlapping halves — no memoisation needed. Fibonacci recomputes fib(3) in every branch — memoisation cuts O(2ⁿ) → O(n).
  • Principle of OptimalityExtending an optimal solution never hurts. If a choice made at step i affects what's optimal at step j, greedy fails — DP tries all choices.
💡 Greedy makes the locally best choice and never revisits. DP tries every choice and keeps the best. If you can prove greedy is correct (exchange argument), skip DP.

State Design
Designing dp[…] — The Core Skill
  1. Name what changesEach variable that changes across subproblems = one dimension of your array.
  2. Define dp[i] preciselyWrite it as a sentence: "dp[i][j] = min cost to convert s[0..i] into t[0..j]."
  3. Find the recurrenceAsk: what was the last decision? Enumerate all choices for it.
  4. Identify base casesEmpty input, index 0/1, capacity 0 — what is trivially known?
  5. Determine orderSubproblems must be solved before they're referenced.
  6. Extract the answerSometimes dp[n], sometimes max over all dp[i], sometimes dp[n][0] vs dp[n][1].
Java Template
Top-Down Memoization (Java)
class Solution { private int[][] memo; public int solve(int[] nums) { int n = nums.length; memo = new int[n][CAPACITY + 1]; for (int[] row : memo) Arrays.fill(row, -1); // -1 = not computed return dp(0, CAPACITY, nums); } private int dp(int i, int cap, int[] nums) { if (i == nums.length) return 0; // base case if (memo[i][cap] != -1) return memo[i][cap]; // cache hit int skip = dp(i + 1, cap, nums); int take = (cap >= nums[i]) ? nums[i] + dp(i + 1, cap - nums[i], nums) : Integer.MIN_VALUE; return memo[i][cap] = Math.max(skip, take); } }
Use Integer.MIN_VALUE / 2 (not MIN_VALUE) for "impossible" — avoids overflow when adding to it.
Java Template
Bottom-Up Tabulation (Java)
public int knapsack(int[] wt, int[] val, int W) { int n = wt.length; // dp[i][w] = max value using items 0..i-1 with capacity w int[][] dp = new int[n + 1][W + 1]; for (int i = 1; i <= n; i++) { for (int w = 0; w <= W; w++) { dp[i][w] = dp[i-1][w]; // skip item i if (wt[i-1] <= w) dp[i][w] = Math.max(dp[i][w], dp[i-1][w - wt[i-1]] + val[i-1]); } } return dp[n][W]; } // Space-optimised: rolling 1D array (reverse loop = 0/1) public int knapsack1D(int[] wt, int[] val, int W) { int[] dp = new int[W + 1]; for (int i = 0; i < wt.length; i++) for (int w = W; w >= wt[i]; w--) // ← reverse! dp[w] = Math.max(dp[w], dp[w - wt[i]] + val[i]); return dp[W]; }

// House Robber — O(n) time, O(1) space public int rob(int[] nums) { int prev2 = 0, prev1 = 0; for (int x : nums) { int curr = Math.max(prev1, prev2 + x); prev2 = prev1; prev1 = curr; } return prev1; } // Kadane's Algorithm — max subarray sum public int maxSubArray(int[] nums) { int curr = nums[0], best = nums[0]; for (int i = 1; i < nums.length; i++) { curr = Math.max(nums[i], curr + nums[i]); best = Math.max(best, curr); } return best; } // Decode Ways — careful with '0' public int numDecodings(String s) { int n = s.length(); int prev2 = 1, prev1 = s.charAt(0) != '0' ? 1 : 0; for (int i = 2; i <= n; i++) { int curr = 0; int one = s.charAt(i-1) - '0'; int two = Integer.parseInt(s.substring(i-2, i)); if (one >= 1) curr += prev1; if (two >= 10 && two <= 26) curr += prev2; prev2 = prev1; prev1 = curr; } return prev1; }

Java Code
Edit Distance + LIS O(n log n)
// Edit Distance — O(mn) time, O(n) space public int minDistance(String word1, String word2) { int m = word1.length(), n = word2.length(); int[] dp = new int[n + 1]; for (int j = 0; j <= n; j++) dp[j] = j; // base: "" → t[0..j] for (int i = 1; i <= m; i++) { int prev = dp[0]; dp[0] = i; for (int j = 1; j <= n; j++) { int temp = dp[j]; if (word1.charAt(i-1) == word2.charAt(j-1)) dp[j] = prev; else dp[j] = 1 + Math.min(prev, Math.min(dp[j], dp[j-1])); prev = temp; } } return dp[n]; } // LIS — O(n log n) using patience sorting public int lengthOfLIS(int[] nums) { int[] tails = new int[nums.length]; int size = 0; for (int x : nums) { // binary search: first tail >= x int lo = 0, hi = size; while (lo < hi) { int mid = (lo + hi) / 2; if (tails[mid] < x) lo = mid + 1; else hi = mid; } tails[lo] = x; if (lo == size) size++; } return size; } // Longest Palindromic Subsequence public int longestPalindromeSubseq(String s) { int n = s.length(); int[][] dp = new int[n][n]; for (int i = 0; i < n; i++) dp[i][i] = 1; for (int len = 2; len <= n; len++) { for (int i = 0; i <= n - len; i++) { int j = i + len - 1; if (s.charAt(i) == s.charAt(j)) dp[i][j] = dp[i+1][j-1] + 2; else dp[i][j] = Math.max(dp[i+1][j], dp[i][j-1]); } } return dp[0][n-1]; }

// 0/1 Knapsack — 1D optimised, O(nW) public int knapsack01(int[] wt, int[] val, int W) { int[] dp = new int[W + 1]; for (int i = 0; i < wt.length; i++) for (int w = W; w >= wt[i]; w--) // ← reverse dp[w] = Math.max(dp[w], dp[w - wt[i]] + val[i]); return dp[W]; } // Coin Change — min coins (unbounded) public int coinChange(int[] coins, int amount) { int[] dp = new int[amount + 1]; Arrays.fill(dp, amount + 1); // "infinity" dp[0] = 0; for (int c : coins) for (int j = c; j <= amount; j++) // ← forward dp[j] = Math.min(dp[j], dp[j - c] + 1); return dp[amount] > amount ? -1 : dp[amount]; } // Count combinations (NOT permutations): outer=coins public int change(int amount, int[] coins) { int[] dp = new int[amount + 1]; dp[0] = 1; for (int c : coins) // ← outer loop = coins for (int j = c; j <= amount; j++) dp[j] += dp[j - c]; return dp[amount]; } // Partition Equal Subset Sum public boolean canPartition(int[] nums) { int sum = 0; for (int x : nums) sum += x; if (sum % 2 != 0) return false; int target = sum / 2; boolean[] dp = new boolean[target + 1]; dp[0] = true; for (int x : nums) for (int j = target; j >= x; j--) // ← reverse 0/1 dp[j] |= dp[j - x]; return dp[target]; }

// Matrix Chain Multiplication — O(n³) public int matrixChain(int[] dims) { int n = dims.length - 1; // n matrices int[][] dp = new int[n][n]; // dp[i][i] = 0 (single matrix, no cost) for (int len = 2; len <= n; len++) { // length of chain for (int i = 0; i <= n - len; i++) { int j = i + len - 1; dp[i][j] = Integer.MAX_VALUE; for (int k = i; k < j; k++) { // split point int cost = dp[i][k] + dp[k+1][j] + dims[i] * dims[k+1] * dims[j+1]; dp[i][j] = Math.min(dp[i][j], cost); } } } return dp[0][n-1]; } // Burst Balloons — O(n³). Pad with 1s at boundaries. public int maxCoins(int[] nums) { int n = nums.length + 2; int[] a = new int[n]; a[0] = a[n-1] = 1; for (int i = 0; i < nums.length; i++) a[i+1] = nums[i]; int[][] dp = new int[n][n]; // dp[i][j]: max coins in open (i,j) for (int len = 2; len < n; len++) { for (int i = 0; i < n - len; i++) { int j = i + len; for (int k = i+1; k < j; k++) { // k = last balloon burst dp[i][j] = Math.max(dp[i][j], dp[i][k] + dp[k][j] + a[i]*a[k]*a[j]); } } } return dp[0][n-1]; }

// Binary Tree Max Path Sum — O(n) class Solution { int ans = Integer.MIN_VALUE; public int maxPathSum(TreeNode root) { dfs(root); return ans; } private int dfs(TreeNode node) { if (node == null) return 0; int L = Math.max(0, dfs(node.left)); // discard negatives int R = Math.max(0, dfs(node.right)); ans = Math.max(ans, L + R + node.val); // path through node return node.val + Math.max(L, R); // return single branch } } // House Robber III — rob/skip pair trick public int rob(TreeNode root) { int[] res = dfs(root); return Math.max(res[0], res[1]); } // returns [rob_this_node, skip_this_node] private int[] dfs(TreeNode node) { if (node == null) return new int[]{0, 0}; int[] L = dfs(node.left); int[] R = dfs(node.right); // rob: take this node → can't take children int rob = node.val + L[1] + R[1]; // skip: don't take this node → children choose freely int skip = Math.max(L[0], L[1]) + Math.max(R[0], R[1]); return new int[]{rob, skip}; }

// TSP — O(n² * 2ⁿ) time, O(n * 2ⁿ) space public int tsp(int[][] dist) { int n = dist.length; int FULL = (1 << n) - 1; // dp[mask][i] = min cost to visit cities in mask, ending at i int[][] dp = new int[1 << n][n]; for (int[] row : dp) Arrays.fill(row, Integer.MAX_VALUE / 2); dp[1][0] = 0; // start at city 0, mask=0b0001 for (int mask = 1; mask <= FULL; mask++) { for (int u = 0; u < n; u++) { if (((mask >> u) & 1) == 0) continue; // u not in mask if (dp[mask][u] == Integer.MAX_VALUE / 2) continue; for (int v = 0; v < n; v++) { if (((mask >> v) & 1) == 1) continue; // v already visited int nextMask = mask | (1 << v); dp[nextMask][v] = Math.min( dp[nextMask][v], dp[mask][u] + dist[u][v] ); } } } int ans = Integer.MAX_VALUE; for (int i = 1; i < n; i++) ans = Math.min(ans, dp[FULL][i] + dist[i][0]); // return home return ans; } // Assign tasks to workers — dp[mask] = min cost to // assign first popcount(mask) workers to tasks in mask public int assignTasks(int[][] cost) { int n = cost.length; int[] dp = new int[1 << n]; Arrays.fill(dp, Integer.MAX_VALUE); dp[0] = 0; for (int mask = 1; mask < (1 << n); mask++) { int worker = Integer.bitCount(mask) - 1; // 0-indexed worker for (int task = 0; task < n; task++) { if (((mask >> task) & 1) == 0) continue; int prev = mask ^ (1 << task); if (dp[prev] != Integer.MAX_VALUE) dp[mask] = Math.min(dp[mask], dp[prev] + cost[worker][task]); } } return dp[(1 << n) - 1]; }

Pattern 7 — Digit DP

Count numbers in [0, N]
Framework
State Dimensions
Count integers in [lo, hi] satisfying a property. Compute f(hi) − f(lo−1). Always recurse over digit positions.
StateTypePurpose
posint 0..D-1Current digit being placed (D = # of digits in N)
tightbooleanAre we still bounded by N's digits? Once false → any digit 0-9 allowed
startedbooleanHave we placed a non-zero digit yet? Handles leading-zero edge cases
extraproblem-specificDigit sum, count of 1s, remainder mod k, last digit placed, etc.
Cache key = (pos, tight, started, extra). With tight=false+started=true, the state is fully determined — cache these freely. With tight=true, don't cache (or cache separately).
Common problems: count numbers with digit sum S, count numbers with no consecutive same digits, count numbers ≡ k (mod m), count step numbers.
Java Code
Digit DP — Count Numbers with Digit Sum ≤ S
class DigitDP { int[] digits; int[][][] memo; // [pos][tight][digitSum] int target; public int countUpTo(long n, int S) { String s = Long.toString(n); digits = new int[s.length()]; for (int i = 0; i < s.length(); i++) digits[i] = s.charAt(i) - '0'; target = S; // memo[pos][tight][sum] — -1 = unvisited memo = new int[digits.length][2][S + 1]; for (int[][] a : memo) for (int[] b : a) Arrays.fill(b, -1); return dp(0, true, false, 0); } private int dp(int pos, boolean tight, boolean started, int sum) { if (sum > target) return 0; // pruning if (pos == digits.length) return started ? 1 : 0; int ti = tight ? 1 : 0; if (!tight && started && memo[pos][ti][sum] != -1) return memo[pos][ti][sum]; // cache hit int limit = tight ? digits[pos] : 9; int result = 0; for (int d = 0; d <= limit; d++) { result += dp( pos + 1, tight && (d == limit), started || (d != 0), started ? sum + d : (d == 0 ? 0 : d) // skip leading zeros ); } if (!tight && started) memo[pos][ti][sum] = result; return result; } }

// Grid Min Path Sum — O(mn), O(n) space public int minPathSum(int[][] grid) { int m = grid.length, n = grid[0].length; int[] dp = grid[0].clone(); for (int j = 1; j < n; j++) dp[j] += dp[j-1]; // first row for (int i = 1; i < m; i++) { dp[0] += grid[i][0]; for (int j = 1; j < n; j++) dp[j] = grid[i][j] + Math.min(dp[j], dp[j-1]); } return dp[n-1]; } // Longest Increasing Path in Matrix — DFS + memo class LIPSolution { int[][] memo, matrix; int m, n; int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}}; public int longestIncreasingPath(int[][] mat) { m = mat.length; n = mat[0].length; matrix = mat; memo = new int[m][n]; int ans = 0; for (int i = 0; i < m; i++) for (int j = 0; j < n; j++) ans = Math.max(ans, dfs(i, j)); return ans; } private int dfs(int i, int j) { if (memo[i][j] != 0) return memo[i][j]; // cache hit memo[i][j] = 1; for (int[] d : dirs) { int ni = i+d[0], nj = j+d[1]; if (ni>=0 && ni=0 && nj matrix[i][j]) memo[i][j] = Math.max(memo[i][j], dfs(ni,nj)+1); } return memo[i][j]; } }

Advanced Optimisations

Competitive / Hard
Optimisation 1
Convex Hull Trick (CHT)
Applies when: dp[i] = min over j<i of: b[j]·a[i] + c[j] — i.e., dp is a min over linear functions. Maintain lower envelope of lines.
  • Lines sorted by slope + queries sortedDeque approach: O(n) total. Pop lines from back when new line dominates.
  • Li Chao Tree (online, unsorted)Segment tree on query values. O(n log n). Handles arbitrary insert/query order.
  • Slope trickSpecial case where dp transition changes the "slope" of a piecewise-linear function. Priority queue on breakpoints.
// CHT deque: add line y=mx+b; query min at x static long[] bad(long[] l1, long[] l2, long[] l3) { // l2 never optimal if intersection(l1,l3) ≤ intersection(l1,l2) return (l3[1]-l1[1])*(l1[0]-l2[0]) <= (l2[1]-l1[1])*(l1[0]-l3[0]); } // Deque of lines {m, b}; query = min(m*x + b)
Optimisation 2
Knuth–Yao Optimisation
Reduces interval DP from O(n³) → O(n²) when the cost function satisfies the quadrangle inequality.
  • Conditioncost(a,c) + cost(b,d) ≤ cost(a,d) + cost(b,c) for a≤b≤c≤d
  • Consequenceopt[i][j-1] ≤ opt[i][j] ≤ opt[i+1][j] — the optimal split point is monotone.
  • Code changeJust bound the k loop: k from opt[i][j-1] to opt[i+1][j]. Everything else stays.
for (int len = 2; len <= n; len++) { for (int i = 0; i <= n - len; i++) { int j = i + len - 1; int lo = opt[i][j-1]; // Knuth bound int hi = (j+1 < n) ? opt[i+1][j] : j; for (int k = lo; k <= hi; k++) { if (dp[i][k] + dp[k+1][j] + cost < dp[i][j]) { dp[i][j] = dp[i][k] + dp[k+1][j] + cost; opt[i][j] = k; } } } }
Optimisation 3
Divide & Conquer Opt.
Reduces O(n²k) DP to O(nk log n) when optimal split point is monotone across rows.
  • Conditionopt[i][j] ≤ opt[i][j+1]: left bound of search range shrinks monotonically
  • AlgorithmFind opt[mid] in [opt_lo..min(opt_hi, mid)], then recurse on halves
  • Apply whenK-partition cost minimisation: split array into k groups minimising sum of costs
void solve(int lo, int hi, int optLo, int optHi) { if (lo > hi) return; int mid = (lo + hi) / 2, opt = optLo; for (int k = optLo; k <= Math.min(optHi, mid - 1); k++) { if (cost(k, mid) < cost(opt, mid)) opt = k; } dp[layer][mid] = cost(opt, mid); solve(lo, mid-1, optLo, opt); solve(mid+1, hi, opt, optHi); }

Space Optimisation

O(n²) → O(n) → O(1)
Technique 1
Rolling Array
When dp[i][j] only depends on dp[i-1][*], keep only two rows. When it only depends on dp[i-1][j-1], dp[i-1][j], dp[i][j-1] — keep one row + one scalar.
// LCS: O(mn) → O(n) with prev row int[] prev = new int[n + 1]; for (char c : s.toCharArray()) { int[] curr = new int[n + 1]; for (int j = 1; j <= n; j++) { if (c == t.charAt(j-1)) curr[j] = prev[j-1] + 1; else curr[j] = Math.max(prev[j], curr[j-1]); } prev = curr; } return prev[n];
Can't reconstruct path with rolling array — keep full table if backtracking needed.
Technique 2
In-Place Update Trick
For Edit Distance: update a single row in-place using a prev scalar to hold the diagonal.
// Edit Distance in O(n) space int[] dp = new int[n + 1]; for (int j = 0; j <= n; j++) dp[j] = j; for (int i = 1; i <= m; i++) { int diag = dp[0]; // save dp[i-1][j-1] dp[0] = i; for (int j = 1; j <= n; j++) { int temp = dp[j]; if (s.charAt(i-1)==t.charAt(j-1)) dp[j] = diag; else dp[j] = 1+Math.min(diag,Math.min(dp[j],dp[j-1])); diag = temp; // advance diagonal } }
Technique 3
Coordinate Compression + BIT
When dp values are large but sparse, compress to ranks and query with BIT or Segment Tree for O(n log n) transitions.
// O(n log n) LIS via BIT on compressed values int[] sorted = nums.clone(); Arrays.sort(sorted); // deduplicate + rank int[] bit = new int[n + 2]; int ans = 0; for (int x : nums) { int rank = Arrays.binarySearch(sorted, x) + 1; int best = query(bit, rank - 1) + 1; ans = Math.max(ans, best); update(bit, rank, best, n); } static int query(int[] bit, int i) { int res = 0; for (; i > 0; i -= i & -i) res = Math.max(res, bit[i]); return res; } static void update(int[] bit, int i, int v, int n) { for (; i <= n; i += i & -i) bit[i] = Math.max(bit[i], v); }

Interview Mindset

Process + Traps
5-Step Approach
Solving Any DP in Interview
  1. Define dp[i] in plain EnglishWrite it out before coding: "dp[i][w] = max value using items 0..i-1 with capacity w." If you can't state it clearly, you don't understand the problem yet.
  2. Find the recurrence from last-step decompositionAsk: what was the LAST decision made to reach this state? What choices existed? Each choice = one case in the recurrence.
  3. Identify base cases preciselyEmpty input (i=0), zero capacity (w=0), single character. Test mentally: dp[0][*] and dp[*][0].
  4. Determine iteration orderEvery subproblem you reference must already be computed. Draw a small 2D table and trace the fill order if unsure.
  5. Code → Verify → Optimise spaceGet a correct O(n²) solution first, then ask: "Do I need all rows, or just the previous one?" Optimise only after correctness is confirmed.
7 Common Mistakes
Where Candidates Fail
  • Off-by-one in base casesIndex 0 = empty prefix, index 1 = first element. Be consistent. Don't mix 0-indexed arrays with 1-indexed dp.
  • Wrong direction for knapsack inner loopForward = item reusable (coin change). Reverse = item used once (partition). This mistake produces wrong answers silently.
  • Answer isn't always dp[n]Sometimes max(dp[i]) for all i (LIS, max subarray). Sometimes dp[n][0] vs dp[n][1] (house robber circle). Always check.
  • Not handling -1 cache correctly in JavaArrays.fill(memo, -1) only works if 0 and positives are valid answers — if -1 is a valid answer, use a separate visited[][] boolean.
  • Integer overflow in cost accumulationsUse long when multiplying indices. Use Integer.MAX_VALUE/2 (not MAX_VALUE) as infinity — adding to MAX_VALUE overflows to negative.
  • Initialising dp with wrong valueFor min problems initialise with Integer.MAX_VALUE/2. For max problems use Integer.MIN_VALUE/2. For count problems use 0.
  • Stack overflow in deep recursionJava default stack is ~500k frames. For n=10⁵ with memoization → switch to tabulation or increase stack with new Thread(null, ()->{...}, "solver", 1<<26).
