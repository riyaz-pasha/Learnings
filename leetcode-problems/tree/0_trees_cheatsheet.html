Terminology & Properties

📐 Core Terms
  • Node — element with value + children
  • Root — top node (no parent)
  • Leaf — node with no children
  • Internal Node — non-leaf node
  • Edge — link between parent & child
  • Sibling — nodes sharing same parent
  • Subtree — node + all descendants
  • Path — sequence of nodes (no repeat)
  • Ancestor — any node on path to root
  • Descendant — any node in subtree
📏 Height vs Depth
⚠️ Interview Trap!
Height and Depth are opposite directions. Know which one the problem asks for.
  • Depth of node = edges from root to node (root = 0)
  • Height of node = edges from node to furthest leaf
  • Height of tree = height of root node
  • Level = Depth + 1 (root at level 1)
  • Leaf node height = 0
  • NULL node height = -1
1 ← depth 0, height 2 / \ 2 3 ← depth 1, height 1 / \ 4 5 ← depth 2, height 0 (leaves)
🔢 Key Formulas
  • Max nodes at level i: 2ⁱ
  • Max nodes in tree of height h: 2^(h+1) - 1
  • Min height for n nodes: ⌊log₂n⌋
  • Max height for n nodes: n-1 (skewed)
  • Leaves in full binary tree: (n+1)/2
  • Internal nodes: leaves - 1
  • Edges in tree: n - 1
  • Children in complete binary tree: ⌊n/2⌋ internal nodes
🌳 Tree Classification (Important!)
TypeDefinitionKey Property

💾 Node Structure

02

Tree Types

🔍 Binary Search Tree (BST)
  • Left subtree contains ONLY nodes < root
  • Right subtree contains ONLY nodes > root
  • Both subtrees are also BSTs
  • In-order traversal gives SORTED order ← crucial!
  • Duplicate handling — clarify with interviewer!
8 / \ 3 10 / \ \ 1 6 14 / \ / 4 7 13
Key invariant: For any node X, ALL nodes in left subtree < X.val AND ALL nodes in right subtree > X.val. Not just immediate children!
📦 AVL Tree
  • Self-balancing BST
  • Balance factor (BF) = h(left) - h(right)
  • BF must be -1, 0, or +1 for every node
  • 4 rotations: LL, RR, LR, RL
  • Stricter balance than Red-Black → faster lookups
  • More rotations on insert/delete → slower writes
LL Rotation (right rotate): Insert into left child of left subtree
RR Rotation (left rotate): Insert into right child of right subtree
LR Rotation: Left rotate left child, then right rotate node
RL Rotation: Right rotate right child, then left rotate node
🔴 Red-Black Tree
  • Every node is Red or Black
  • Root is always Black
  • NULL leaves are Black
  • Red node → both children must be Black
  • All paths from node to NULL have same black-node count (Black Height)
  • Height ≤ 2 log(n+1) — looser than AVL
  • Used in: Java TreeMap, C++ std::map, Linux scheduler
AVL vs RB: AVL = read-heavy (faster lookup), Red-Black = write-heavy (fewer rotations). std::map uses Red-Black.
🔠 Trie (Prefix Tree)
  • Each edge represents a character
  • Root represents empty string
  • Path from root to node = prefix
  • Insert/Search/Delete: O(L) where L = word length
  • Space: O(alphabet_size × L × n) — can be large
  • Use when: autocomplete, spell check, prefix queries, word search
class TrieNode: def __init__(self): self.children = {} # or [None]*26 self.is_end = False
📈 Segment Tree
  • Range queries (sum, min, max) in O(log n)
  • Point updates in O(log n)
  • Array-based: node i → children 2i, 2i+1
  • Space: O(4n) for size-n array
  • Lazy propagation for range updates
  • Use when: repeated range queries + point/range updates
Node index: Root=1, left child=2i, right child=2i+1, parent=i//2. Array size = 4*n to be safe.
📊 Fenwick Tree (BIT)
  • Prefix sum queries in O(log n)
  • Point updates in O(log n)
  • Simpler to implement than Segment Tree
  • Uses: lowbit = i & (-i)
  • Cannot do arbitrary range queries (only prefix)
  • 1-indexed array (index 0 unused)
def update(bit, i, delta, n): while i <= n: bit[i] += delta i += i & (-i) # add lowbit def query(bit, i): s = 0 while i > 0: s += bit[i] i -= i & (-i) # remove lowbit return s
🌲 Heap (Priority Queue)
  • Complete Binary Tree stored as array
  • Max-Heap: parent ≥ children
  • Min-Heap: parent ≤ children
  • Node i: left=2i+1, right=2i+2, parent=(i-1)//2
  • Insert/Delete: O(log n); Peek: O(1); Build: O(n)
  • heapq in Python = Min-Heap by default
  • Max-heap: insert -val, negate on pop
Heap is NOT BST! Heap only guarantees parent-child order. No ordering between siblings.
🌿 B-Tree / B+ Tree
  • Self-balancing, multi-way tree
  • All leaves at same level
  • Order m: each node has at most m children, ⌈m/2⌉ min children
  • Used in: databases, file systems (disk-friendly)
  • B+ Tree: only leaves store data; internal nodes store only keys
  • B+ Tree leaves linked → efficient range queries
  • Rarely implemented in interviews — just know the concept

Tree Traversals

Quick Memory Aid: In Pre/In/Post — the "pre/in/post" refers to where the ROOT is processed relative to children. Pre = Root first, In = Root middle, Post = Root last. Left always before Right (by convention).
🔄 Inorder (L → Root → R)
When to use: BST problems (gives sorted order), problems needing sorted sequence, kth smallest/largest in BST
Recursive
def inorder(root): if not root: return [] return inorder(root.left) + [root.val] + inorder(root.right)
Iterative (with stack) — more important!
def inorder_iter(root): stack, res, curr = [], [], root while curr or stack: while curr: # go left as far as possible stack.append(curr) curr = curr.left curr = stack.pop() # process node res.append(curr.val) curr = curr.right # move to right return res
BST Inorder = Sorted array. This unlocks: validate BST, recover BST, kth smallest, etc.
🔄 Preorder (Root → L → R)
When to use: Serialize/copy tree, path problems, print directory structure, reconstruct tree from traversal
Recursive
def preorder(root): if not root: return [] return [root.val] + preorder(root.left) + preorder(root.right)
Iterative
def preorder_iter(root): if not root: return [] stack, res = [root], [] while stack: node = stack.pop() res.append(node.val) if node.right: stack.append(node.right) # push right first if node.left: stack.append(node.left) return res
Preorder is great for serialization because you can reconstruct tree from it (given it's a BST).
🔄 Postorder (L → R → Root)
When to use: Delete tree, evaluate expression tree, bottom-up calculations (height, diameter), dependency resolution
Recursive
def postorder(root): if not root: return [] return postorder(root.left) + postorder(root.right) + [root.val]
Iterative (trick: reverse of modified preorder)
def postorder_iter(root): if not root: return [] stack, res = [root], [] while stack: node = stack.pop() res.append(node.val) if node.left: stack.append(node.left) # push left first if node.right: stack.append(node.right) return res[::-1] # reverse at end
🌊 Level Order (BFS)
When to use: Level-by-level processing, shortest path in unweighted tree, connect level nodes, right side view, zigzag traversal
Level-aware BFS (most useful form)
from collections import deque def level_order(root): if not root: return [] queue, res = deque([root]), [] while queue: level = [] for _ in range(len(queue)): # fix snapshot of level size! node = queue.popleft() level.append(node.val) if node.left: queue.append(node.left) if node.right: queue.append(node.right) res.append(level) return res
Key trick: for _ in range(len(queue)) BEFORE adding children captures the current level's size. This is the #1 BFS interview pattern.
🔀 Morris Traversal (O(1) Space Inorder — No Stack/Recursion)

Temporarily modifies tree. Uses threaded binary tree concept. Important for space-constrained interviews.

def morris_inorder(root): curr, res = root, [] while curr: if not curr.left: res.append(curr.val) # no left → process and go right curr = curr.right else: pre = curr.left while pre.right and pre.right != curr: pre = pre.right # find inorder predecessor if not pre.right: # first visit: create thread pre.right = curr curr = curr.left else: # second visit: remove thread pre.right = None res.append(curr.val) curr = curr.right return res

BST Operations

🔍 BST Search
def search_bst(root, val): if not root or root.val == val: return root if val < root.val: return search_bst(root.left, val) return search_bst(root.right, val)
➕ BST Insert
def insert_bst(root, val): if not root: return TreeNode(val) if val < root.val: root.left = insert_bst(root.left, val) elif val > root.val: root.right = insert_bst(root.right, val) return root # always return root!
🗑️ BST Delete (Tricky!)
3 Cases: (1) Node is leaf → remove it. (2) Node has 1 child → replace with child. (3) Node has 2 children → replace with inorder successor (leftmost in right subtree) or inorder predecessor.
def delete_bst(root, key): if not root: return None if key < root.val: root.left = delete_bst(root.left, key) elif key > root.val: root.right = delete_bst(root.right, key) else: # found the node if not root.left: return root.right if not root.right: return root.left # has 2 children → find inorder successor succ = root.right while succ.left: succ = succ.left root.val = succ.val root.right = delete_bst(root.right, succ.val) return root
✅ Validate BST
Common Mistake: Only comparing node with immediate children is WRONG. You must check against entire valid range.
def isValidBST(root, lo=float('-inf'), hi=float('inf')): if not root: return True if not (lo < root.val < hi): return False return (isValidBST(root.left, lo, root.val) and isValidBST(root.right, root.val, hi))
Alternatively: inorder traversal → check if resulting array is strictly increasing.
🏆 Kth Smallest in BST
# Iterative inorder — stop at kth element def kthSmallest(root, k): stack, curr = [], root while curr or stack: while curr: stack.append(curr) curr = curr.left curr = stack.pop() k -= 1 if k == 0: return curr.val curr = curr.right
🔎 Inorder Successor / Predecessor
Successor = smallest value > node. Predecessor = largest value < node.
def inorder_successor(root, p): succ = None while root: if p.val < root.val: succ = root # potential successor root = root.left else: root = root.right return succ # Rule: successor = leftmost of right subtree (if exists) # else = lowest ancestor where node is in left subtree

Balance Check & Conversion

⚖️ Check Height-Balanced

A tree is height-balanced if for every node, |height(left) - height(right)| ≤ 1.

# O(n) — single pass, returns -1 if unbalanced def isBalanced(root) -> bool: def height(node): if not node: return 0 lh = height(node.left) if lh == -1: return -1 # short-circuit rh = height(node.right) if rh == -1: return -1 if abs(lh - rh) > 1: return -1 return max(lh, rh) + 1 return height(root) != -1
Pattern: Return -1 as sentinel for "unbalanced" — avoids a second pass. This is the optimal O(n) approach.
🏗️ Sorted Array → Balanced BST
When used: "Convert sorted array to BST", database index construction
def sortedArrayToBST(nums): if not nums: return None mid = len(nums) // 2 node = TreeNode(nums[mid]) node.left = sortedArrayToBST(nums[:mid]) node.right = sortedArrayToBST(nums[mid+1:]) return node # Same logic for sorted linked list (use slow/fast pointer) def sortedListToBST(head): if not head: return None if not head.next: return TreeNode(head.val) prev, slow, fast = None, head, head while fast and fast.next: prev, slow, fast = slow, slow.next, fast.next.next prev.next = None # cut left half node = TreeNode(slow.val) node.left = sortedListToBST(head) node.right = sortedListToBST(slow.next) return node

Special Algorithms

🔗 Lowest Common Ancestor (LCA)
3 Approaches depending on constraints. Know all three!
# Binary Tree LCA (general) — O(n) def lowestCommonAncestor(root, p, q): if not root or root in (p, q): return root left = lowestCommonAncestor(root.left, p, q) right = lowestCommonAncestor(root.right, p, q) if left and right: return root # p,q in diff subtrees return left or right # BST LCA — O(log n) — use BST property! def lcaBST(root, p, q): while root: if p.val < root.val and q.val < root.val: root = root.left elif p.val > root.val and q.val > root.val: root = root.right else: return root # split point = LCA
Why it works: If both p,q are smaller → LCA is in left subtree. If both larger → right. Otherwise current node is LCA (they split here).
📏 Diameter of Binary Tree
Diameter = longest path between any two nodes. Path may or may not go through root!
def diameterOfBinaryTree(root) -> int: self.res = 0 def height(node): if not node: return 0 lh = height(node.left) rh = height(node.right) self.res = max(self.res, lh + rh) # path through node return max(lh, rh) + 1 height(root) return self.res
Gotcha: Diameter is in EDGES. The number of edges = nodes_in_path - 1. lh+rh already gives edges count if height returns 0 for null.
🌟 Max Path Sum
Tricky: Values can be negative. A path must have at least one node. Max contribution from a subtree = max(0, recursive_result) to allow "not taking" a subtree.
def maxPathSum(root) -> int: res = [float('-inf')] def dfs(node): if not node: return 0 left = max(dfs(node.left), 0) # ignore negative right = max(dfs(node.right), 0) res[0] = max(res[0], node.val + left + right) return node.val + max(left, right) # can only extend one way dfs(root) return res[0]
📡 Serialize & Deserialize
class Codec: def serialize(self, root): vals = [] def dfs(node): if not node: vals.append('#'); return vals.append(str(node.val)) dfs(node.left); dfs(node.right) dfs(root) return ','.join(vals) def deserialize(self, data): vals = iter(data.split(',')) def build(): v = next(vals) if v == '#': return None node = TreeNode(int(v)) node.left, node.right = build(), build() return node return build()

Patterns & Interview Cues

How to think: When you see a tree problem, immediately ask: (1) Binary or N-ary? (2) BST or generic binary tree? (3) Does order matter? (4) Single-pass or multi-pass? (5) Top-down or bottom-up? (6) Need parent pointer?
🔻 Pattern 1: Recursive DFS with Return Value Most Common
When: Need to compute something for each subtree and propagate up. Height, sum, count, diameter, path sum. Think "what does this function return?" and "what do I do with left and right return values?"
Cues: "height of", "depth of", "sum of", "count nodes", "check symmetric", "check balanced", "max/min value", "longest path"
Template
def solve(node): if not node: return base_case left_val = solve(node.left) right_val = solve(node.right) # combine left_val, right_val, node.val return result
Examples
# Height return max(lh, rh) + 1 # Node Count return left + right + 1 # Sum of all nodes return left + right + node.val # Is symmetric? return isMirror(l.left,r.right) and \ isMirror(l.right,r.left)
🔽 Pattern 2: Top-Down DFS (Pass state down) Very Common
When: Decision depends on context from ancestors. "Is this path valid?", "Is this sum achievable?", accumulate something along root-to-leaf path.
Cues: "root-to-leaf path", "path sum equals target", "paths from root", "prefix at current depth"
def solve(node, state_from_parent): if not node: return new_state = combine(state_from_parent, node.val) if is_leaf(node): check_result(new_state) solve(node.left, new_state) solve(node.right, new_state) # Path Sum Example def hasPathSum(node, remaining): if not node: return False remaining -= node.val if not node.left and not node.right: return remaining == 0 return hasPathSum(node.left, remaining) or \ hasPathSum(node.right, remaining)
🌊 Pattern 3: BFS Level Order Level problems
When: Need level-by-level info, minimum depth, right/left view, level averages, zigzag traversal, connect siblings at same level, tree width.
Cues: "level", "row", "minimum depth", "right side view", "bottom-up level order", "zigzag", "same level neighbors"
Gotcha: For right side view → take LAST element of each level. For left view → take FIRST. For level averages → sum/count per level. Always use for _ in range(len(queue)) to process level by level.
📍 Pattern 4: Path Tracking (Backtracking) Path Problems
When: Need to collect ALL valid root-to-leaf paths. Track current path, add to result at leaf, backtrack by removing last element.
def allPaths(root): res, path = [], [] def dfs(node): if not node: return path.append(node.val) if not node.left and not node.right: res.append(list(path)) # copy! not reference dfs(node.left) dfs(node.right) path.pop() # backtrack dfs(root) return res
Gotcha: Always list(path) not path when appending to results. The path list is shared across calls!
🔢 Pattern 5: Count Nodes / Check Conditions Structural
When: Count leaves, count nodes matching condition, check if two trees are identical, check subtree relationship.
# Count Good Nodes (value ≥ all values on path from root) def goodNodes(root, max_so_far=float('-inf')) -> int: if not root: return 0 good = 1 if root.val >= max_so_far else 0 new_max = max(max_so_far, root.val) return good + goodNodes(root.left, new_max) + \ goodNodes(root.right, new_max)
🗂️ Pattern 6: BST-Specific (Use Sorted Property) BST Only
When: Problem explicitly says BST. Always exploit the ordering property. Inorder gives sorted sequence. Range queries are O(log n) not O(n).
Key insight: BST → think binary search. If target < current, go left; if > current, go right. This eliminates half the tree at each step.
# BST Range Sum (sum all in [lo, hi]) def rangeSumBST(root, lo, hi): if not root: return 0 if root.val < lo: # prune left return rangeSumBST(root.right, lo, hi) if root.val > hi: # prune right return rangeSumBST(root.left, lo, hi) return root.val + \ rangeSumBST(root.left, lo, hi) + \ rangeSumBST(root.right, lo, hi)
# Two Sum in BST — O(n) time, O(n) space def findTarget(root, k): seen = set() def dfs(node): if not node: return False if k - node.val in seen: return True seen.add(node.val) return dfs(node.left) or dfs(node.right) return dfs(root)

DFS Deep Dive

📐 DFS: Diameter & Path through Node
Key pattern: When a path CAN go through a node (using both children), you update global result. But you can only RETURN one branch to parent.
# Applies to: diameter, max path sum, LCA def dfs(node): if not node: return 0 # or -inf for sum left = dfs(node.left) right = dfs(node.right) # Path THROUGH this node (both children) update_global(left + right + node.val) # Return: can only extend ONE direction return max(left, right) + node.val
💡 DFS: Global State via Closure / List

When inner recursive function needs to update a value, use a list [0] or nonlocal.

# Method 1: List as mutable container res = [0] def dfs(node): res[0] = max(res[0], ...) # Method 2: nonlocal (Python 3+) res = 0 def dfs(node): nonlocal res res = max(res, ...) # Method 3: self attribute (in class) self.res = 0 def dfs(node): self.res = max(self.res, ...)
🔢 DFS: Build Number Along Path
Pattern: Sum Root to Leaf Numbers, Sum of Paths as Binary/Decimal, etc.
# Sum Root to Leaf Numbers def sumNumbers(root) -> int: def dfs(node, curr): if not node: return 0 curr = curr * 10 + node.val if not node.left and not node.right: return curr return dfs(node.left, curr) + \ dfs(node.right, curr) return dfs(root, 0)
✂️ DFS: Prune / Modify Tree
When: Remove leaves with value 0, flatten tree, prune subtrees not containing target, delete nodes in list.
# Prune subtrees with no 1-valued nodes def pruneTree(root): if not root: return None root.left = pruneTree(root.left) root.right = pruneTree(root.right) # return None if leaf and value is 0 if not root.left and not root.right \ and root.val == 0: return None return root
Key trick: Post-order! Process children FIRST, then decide whether to keep current node. If you prune top-down, you might miss bottom conditions.

BFS Patterns

👁️ Right / Left Side View
def rightSideView(root): if not root: return [] queue, res = deque([root]), [] while queue: level_size = len(queue) for i in range(level_size): node = queue.popleft() if i == level_size - 1: # last of level res.append(node.val) if node.left: queue.append(node.left) if node.right: queue.append(node.right) return res # DFS alternative for right view: def rightView_DFS(root, level=0, res=None): if res is None: res = [] if not root: return res if level == len(res): res.append(root.val) # first at this level rightView_DFS(root.right, level+1, res) # right first! rightView_DFS(root.left, level+1, res) return res
↔️ Zigzag Level Order
def zigzagLevelOrder(root): if not root: return [] queue, res, left_to_right = deque([root]), [], True while queue: level = [] for _ in range(len(queue)): node = queue.popleft() level.append(node.val) if node.left: queue.append(node.left) if node.right: queue.append(node.right) res.append(level if left_to_right else level[::-1]) left_to_right = not left_to_right return res
🔗 Connect Next Right Pointers
Perfect binary tree: O(1) space using existing next pointers. General tree: use BFS O(n) space.
# Perfect Binary Tree — O(1) space def connect(root): if not root: return None leftmost = root while leftmost.left: # for each level curr = leftmost while curr: curr.left.next = curr.right if curr.next: curr.right.next = curr.next.left curr = curr.next leftmost = leftmost.left return root
📏 Min Depth (BFS is optimal!)
BFS > DFS here: BFS finds first leaf = minimum depth. DFS must visit all nodes to find minimum. BFS is O(min_depth), DFS is O(n).
def minDepth(root) -> int: if not root: return 0 queue = deque([(root, 1)]) while queue: node, depth = queue.popleft() if not node.left and not node.right: return depth # first leaf = minimum! if node.left: queue.append((node.left, depth+1)) if node.right: queue.append((node.right, depth+1))

Advanced Patterns

🔢 Vertical Order Traversal
Approach: BFS/DFS + track (row, col). Sort by col, then row, then value. Use dict col → [(row, val)].
from collections import defaultdict def verticalTraversal(root): cols = defaultdict(list) def dfs(node, row, col): if not node: return cols[col].append((row, node.val)) dfs(node.left, row+1, col-1) dfs(node.right, row+1, col+1) dfs(root, 0, 0) return [[v for _, v in sorted(cols[c])] for c in sorted(cols)]
🌿 Flatten Binary Tree to Linked List
In-place: Right-threading. Preorder result. Use Morris-like approach.
def flatten(root) -> None: curr = root while curr: if curr.left: # find rightmost of left subtree pre = curr.left while pre.right: pre = pre.right # thread: rightmost → curr.right pre.right = curr.right curr.right = curr.left curr.left = None curr = curr.right
🏗️ Construct from Traversals
Need 2 traversals to uniquely reconstruct (unless BST). Inorder+Preorder or Inorder+Postorder. Preorder+Postorder alone is NOT unique for general trees.
# Build from Preorder + Inorder def buildTree(preorder, inorder): if not inorder: return None root_val = preorder[0] # first of preorder = root idx = inorder.index(root_val) node = TreeNode(root_val) # left subtree has idx elements node.left = buildTree(preorder[1:idx+1], inorder[:idx]) node.right = buildTree(preorder[idx+1:], inorder[idx+1:]) return node
Optimize: Use hashmap for inorder index lookups → O(n) instead of O(n²).
🌐 N-ary Tree Traversals
# N-ary Preorder def preorder(root): if not root: return [] res = [root.val] for child in root.children: res.extend(preorder(child)) return res # N-ary Level Order def levelOrder(root): if not root: return [] queue, res = deque([root]), [] while queue: level = [] for _ in range(len(queue)): node = queue.popleft() level.append(node.val) queue.extend(node.children) res.append(level) return res # Max Depth N-ary def maxDepth(root) -> int: if not root: return 0 return 1 + max((maxDepth(c) for c in root.children), default=0)
🔡 Trie: Insert, Search, StartsWith
class Trie: def __init__(self): self.root = TrieNode() def insert(self, word: str) -> None: node = self.root for ch in word: if ch not in node.children: node.children[ch] = TrieNode() node = node.children[ch] node.is_end = True def search(self, word: str) -> bool: node = self.root for ch in word: if ch not in node.children: return False node = node.children[ch] return node.is_end def startsWith(self, prefix: str) -> bool: node = self.root for ch in prefix: if ch not in node.children: return False node = node.children[ch] return True # note: no is_end check!
💰 House Robber III (Tree DP)
Pattern: Tree DP — return (rob_this, skip_this) pair from each node. Classic "take or skip" DP on tree.
def rob(root) -> int: def dfs(node): # returns (rob_root, skip_root) if not node: return (0, 0) l_rob, l_skip = dfs(node.left) r_rob, r_skip = dfs(node.right) # rob this node → must skip children rob = node.val + l_skip + r_skip # skip this → children can rob or skip skip = max(l_rob, l_skip) + max(r_rob, r_skip) return (rob, skip) return max(dfs(root))

Key Algorithms

🔃 Invert / Mirror Tree
def invertTree(root): if not root: return None # swap children first, then recurse root.left, root.right = root.right, root.left invertTree(root.left) invertTree(root.right) return root
🪞 Symmetric Tree
def isSymmetric(root) -> bool: def mirror(left, right): if not left and not right: return True if not left or not right: return False return (left.val == right.val and mirror(left.left, right.right) and mirror(left.right, right.left)) return mirror(root.left, root.right)
🧮 Path Sum III (Prefix Sum + DFS)
Key insight: Use prefix sum hashmap. currentSum - target = a previous prefix sum → valid path exists.
def pathSum(root, target) -> int: from collections import defaultdict count = defaultdict(int) count[0] = 1 # empty prefix def dfs(node, curr_sum): if not node: return 0 curr_sum += node.val res = count[curr_sum - target] count[curr_sum] += 1 res += dfs(node.left, curr_sum) + dfs(node.right, curr_sum) count[curr_sum] -= 1 # backtrack! return res return dfs(root, 0)
🔄 BST → Greater Sum Tree
Trick: Reverse inorder (Right → Root → Left) accumulates suffix sums in a BST.
def convertBST(root): suffix = 0 def reverse_inorder(node): nonlocal suffix if not node: return reverse_inorder(node.right) # visit larger first suffix += node.val node.val = suffix reverse_inorder(node.left) reverse_inorder(root) return root
🔄 Recover BST (Two Swapped Nodes)
Inorder should be sorted. Find two violations. They are the swapped nodes. O(n) time, O(1) space with Morris.
def recoverTree(root) -> None: first = second = prev = None def inorder(node): nonlocal first, second, prev if not node: return inorder(node.left) if prev and prev.val > node.val: if not first: first = prev # first violation second = node # always update second prev = node inorder(node.right) inorder(root) first.val, second.val = second.val, first.val
🌐 Count Complete Tree Nodes (O(log²n))
Trick: Perfect binary tree has 2^h - 1 nodes. Check if left-most depth = right-most depth. If equal → perfect, count directly.
def countNodes(root) -> int: if not root: return 0 lh, rh = 0, 0 l, r = root, root while l: lh += 1; l = l.left while r: rh += 1; r = r.right if lh == rh: # perfect binary tree return 2 ** lh - 1 return 1 + countNodes(root.left) + countNodes(root.right)

Edge Cases & Gotchas

🚨 Always Check
  • Empty tree (root = None)
  • Single node tree
  • All left / all right skewed tree
  • Tree with only root (no children)
  • Negative values in the tree
  • Duplicate values
  • Very deep tree → stack overflow with recursion?
  • k = 1 or k = n in kth-element problems
  • Target node is root
  • p or q is null in LCA problems
⚠️ Common Mistakes
  • Checking only immediate children for BST validity (should check full range)
  • Forgetting to copy path list: res.append(path) → should be list(path)
  • Using root == p instead of root is p when comparing nodes
  • Off-by-one in height: null = -1 or 0?
  • Not handling duplicate keys in BST (clarify with interviewer)
  • Returning -inf for empty sum (needed when all nodes are negative)
  • Assuming input array is sorted when building BST
  • Forgetting return root at end of recursive insert/delete
🔥 Tricky Scenarios
  • Min Depth: node with only one child is NOT a leaf! Both children must be null.
  • Max Path Sum with negative values: never discard a node but can choose to not include its subtree.
  • Diameter can start and end at any node — doesn't go through root necessarily.
  • Path Sum III: paths go downward only (root to any node, not any node to any node).
  • Symmetric check: compare left.left with right.right (not right.left).
  • LCA when a node is ancestor of the other: LCA is the ancestor itself.
  • Level Order bottom-up: just reverse the result of top-down level order.

Interview Strategy

✅ When You See a Tree Problem
1
Clarify the tree type: Binary? BST? N-ary? Can it be null? Can values be negative? Duplicates allowed?
2
Identify traversal needed: Level-by-level → BFS. Top-down accumulation → DFS preorder. Bottom-up calculation → DFS postorder. Sorted order → DFS inorder on BST.
3
Sketch the recursion: What does the function return? What's the base case? What do left and right results tell you?
4
Handle edge cases first: null node, single node, then write the general case.
5
State complexity clearly: O(n) time; O(h) space for DFS where h = height = O(log n) balanced, O(n) worst.

🧠 Mental Models to Internalize
🌳 Tree = Recursion
Every tree problem is "solve for root, then trust recursion solves left and right subtrees." Define what the function returns and what the base case is. The rest follows.
📮 Return value = Message upward
The return value from a recursive call is what the subtree tells the parent. Height, count, sum, existence — design this message carefully. When two pieces of info are needed, return a tuple.
🌐 Global = Track across calls
When the answer might span both subtrees (diameter, max path sum), you need a global tracker. The return value only helps the parent; it can't capture cross-subtree answers.
🎭 Common Follow-Up Questions
  • What if the tree is very deep? → Stack overflow in recursion. Convert to iterative with explicit stack, or use BFS.
  • Can you do it iteratively? → Inorder/Preorder/Postorder all have iterative versions. Know at least inorder iterative cold.
  • Can you do it in O(1) space? → Morris traversal for traversal, O(1) space next-pointer trick for connect-level problems in perfect binary tree.
  • What if we do this frequently? → Cache results (memoization), augment BST with extra info (size, min, max), use balanced BST or segment tree.
  • What if values can repeat? → BST: clarify convention (left ≤ root or left < root). Affects all BST validity checks.
  • What if the tree is an N-ary tree? → Replace node.left, node.right with for child in node.children. Same logic applies.
