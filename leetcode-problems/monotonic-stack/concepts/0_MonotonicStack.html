Group 1 — Next Element (traverse Right → Left)

Next Greater Element

NEXT GREATER
Example
in: [4, 5, 2, 10, 8] out: [5, 10, 10, -1, -1]
4→5 (next bigger) · 5→10 · 2→10 · 10→none · 8→none 
public int[] nextGreaterElement(int[] nums) {
    int n = nums.length;
    int[] result = new int[n];
    // Stack holds INDICES (not values) so we can
    // look up nums[stack.peek()] when needed.
    Stack<Integer> stack = new Stack<>();

    // ◀ AXIS 1: traverse RIGHT → LEFT so that when
    //   we process index i, the stack already holds
    //   candidates to the RIGHT of i.
    for (int i = n - 1; i >= 0; i--) {

        // Discard elements that cannot be the answer for
        // i because they are NOT greater than nums[i].
        while (!stack.isEmpty() && nums[stack.peek()] <= nums[i]) {
            stack.pop(); // pop: ≤ current (GREATER variant)
        }

        // Top of stack (if any) is the first element to
        // the right that is strictly GREATER than nums[i].
        result[i] = stack.isEmpty() ? -1 : nums[stack.peek()];

        stack.push(i); // always push current index
    }
    return result;
}

Group 3 — Circular Next Element (wrap-around, traverse 2n Right → Left)
Circular idea: simulate a doubled array by running 2n iterations using i % n. The second pass (i ≥ n) pre-loads the stack with elements that would appear "to the right after wrap". Only fill result during the first pass (i < n) to avoid double-writing.

Circular Next Greater

NEXT GREATER CIRCULAR
Example
in: [5, 4, 3, 2, 1] out: [-1, 5, 5, 5, 5]
5→none (largest) · 4→5(wrap) · 3→5(wrap) · 2→5(wrap) · 1→5(wrap) 
public int[] nextGreaterElementCircular(int[] nums) {
    int n = nums.length;
    int[] result = new int[n];
    Arrays.fill(result, -1); // default -1; only overwrite in first pass
    Stack<Integer> stack = new Stack<>();

    // ◀ CIRCULAR: loop 2n times instead of n.
    //   Second pass (i ≥ n) fills the stack with
    //   "future" elements for the wrap-around window.
    for (int i = 2 * n - 1; i >= 0; i--) {
        int idx = i % n; // real index into nums[]

        while (!stack.isEmpty() && nums[stack.peek()] <= nums[idx]) {
            stack.pop();
        }

        // ◀ Only write result in first real pass (i < n).
        //   Second pass just primes the stack — no writes.
        if (i < n) {
            result[idx] = stack.isEmpty() ? -1 : nums[stack.peek()];
        }

        stack.push(idx);
    }
    return result;
}

Group 4 — Circular Previous Element (wrap-around, traverse 2n Left → Right)
Circular idea: mirror of group 3. Traverse left → right for 2n steps. The first pass (i < n) pre-loads the stack. Only fill result during the second pass (i ≥ n).

public int[] previousGreaterElementCircular(int[] nums) {
    int n = nums.length;
    int[] result = new int[n];
    Arrays.fill(result, -1);
    Stack<Integer> stack = new Stack<>();

    // ◀ CIRCULAR + PREVIOUS: traverse LEFT → RIGHT for 2n steps.
    //   First pass (i < n) primes the stack.
    //   Second pass (i ≥ n) fills result with circular answers.
    for (int i = 0; i < 2 * n; i++) {
        int idx = i % n;

        while (!stack.isEmpty() && nums[stack.peek()] <= nums[idx]) {
            stack.pop(); // pop: ≤ current (GREATER variant)
        }

        if (i >= n) { // second pass only
            result[idx] = stack.isEmpty() ? -1 : nums[stack.peek()];
        }

        stack.push(idx);
    }
    return result;
}

TopicDetail
Bug fix previousSmallerElement originally had nums[stack.peek()] >= nums[i] (using variable i instead of index). Fixed to use the loop variable consistently.
Loop direction NEXT → traverse right-to-left so right-side candidates are already on the stack.
PREV → traverse left-to-right so left-side candidates are already on the stack.
Pop condition GREATER → pop while top ≤ current (keep the stack monotonically decreasing).
SMALLER → pop while top ≥ current (keep the stack monotonically increasing).
Circular fill window Circular NEXT: loop 2n-1 → 0; fill result only when i < n.
Circular PREV: loop 0 → 2n-1; fill result only when i ≥ n.
Complexity All variants: O(n) time — each element is pushed and popped at most once.
Circular variants: still O(n) — 2n iterations, same amortized argument.
